# Opinion.trade 포인트 시스템 (PTS) 정리

> 출처: [Point System (PTS) | OPINION Docs](https://docs.opinion.trade/incentive-plans/point-system-pts)  
> 이 파일은 공식 문서를 바탕으로 정리한 것입니다.

---

## 1. Opinion 포인트(PTS)가 뭔가요?

**비유**: Opinion 예측 시장에서 "참여 보상"을 위한 점수입니다.  
- 거래·유동성 공급·포지션 보유 등으로 포인트를 쌓으면  
- 매주 분배되는 인센티브 풀에서 내 몫 비율을 정하는 기준이 됩니다.  
- 아직 "1포인트 = N USDT" 같은 고정 환산율은 없고, 비율 기반 분배입니다.

---

## 2. 참여 자격

| 항목 | 내용 |
|------|------|
| **최소 요건** | 총 거래량 $200 이상 (프로그램 참여 자격) |
| **주간 리더보드** | 매주 업데이트 |
| **분배 방식** | 주간 고정 풀에서 비율대로 분배, 초기 참여자가 유리 |

---

## 3. 포인트를 쌓는 방법 (메인넷 기준)

### 3-1. 리밋 오더 (유동성 공급)

| 조건 | 설명 |
|------|------|
| **최소 금액** | $10 USDT 이상 |
| **오더 가격** | 시장 가격에 가까울수록 더 많은 포인트 |
| **유지 시간** | 오더를 오래 열어둘수록 더 많은 포인트 |
| **오더 크기** | 클수록 포인트 증가 |

**예시**: "12월 연준 금리 인하" 시장 확률이 0.20일 때  
- 리밋 매수 0.15 → 적은 포인트  
- 리밋 매수 0.19 → 더 많은 포인트 (가격 발견에 기여)

**요약**: 시장 메이커처럼 유동성을 제공할수록 보상이 큼. 리밋 오더 > 마켓 오더.

### 3-2. 포지션 보유

| 조건 | 설명 |
|------|------|
| **토큰 수** | 보유 수가 많을수록 포인트 증가 |
| **보유 기간** | 보유 기간이 길수록 포인트 증가 |
| **시장별 가중치** | 시장마다 다른 배수가 적용될 수 있음 |

조건부 토큰 포지션을 유지하는 것 자체가 지속적인 보상 대상.

### 3-3. 거래 활동

| 조건 | 설명 |
|------|------|
| **거래 규모** | 거래량이 클수록 비례해 포인트 증가 |
| **의도** | 확신·정보 기여를 반영한 설계 |

---

## 4. 보너스(가중치)가 큰 활동

- **품질 높은 거래**: 시장 정확도·가격 발견에 도움이 되는 활동
- **지정 시장**: Opinion이 지정한 시장은 일반 시장보다 높은 가중치
- **얼리 메이커 유동성**: 초기/신규 시장에서 오더북 형성에 기여

---

## 5. API·구현 관점 정리

### 5-1. 현재 사용 가능한 것

| 항목 | 설명 |
|------|------|
| **거래 내역** | `GET /trade/user/{walletAddress}` – 체결된 거래 목록 |
| **포지션** | `GET /positions/user/{walletAddress}` – 사용자 포지션 목록 |

→ 거래·포지션 **데이터는** API로 가져올 수 있음.

### 5-2. Earned Point 직접 조회 API

- 현재 Opinion 공식 문서에는 **"사용자 누적 포인트"를 직접 반환하는 API**는 보이지 않음.
- PTS 규칙(유동성·보유·거래량 등)은 문서로 공개되어 있지만,  
  실제 포인트 점수는 주간 리더보드·내부 시스템에서 계산·업데이트하는 것으로 보임.

### 5-3. Earned Point를 우리 쪽에서 구현할 수 있을까?

| 방식 | 가능 여부 | 설명 |
|------|-----------|------|
| **공식 포인트 API 사용** | ❌ | 사용자 포인트 조회용 공개 API가 문서에 없음 |
| **공식 포인트 API 신규 요청** | ? | Opinion 측에 포인트 조회 API 요청 필요 |
| **우리 서비스에서 추정** | △ | Trade/Position API 데이터로 PTS 규칙을 **근사** 적용 가능, 공식 값과 차이 발생 가능 |

**추정 구현 시 참고사항**  
- Trade API로 거래량·거래 유형(리밋/마켓) 등 파악  
- Position API로 보유 수·보유 기간 파악  
- 다만 가격과 시장 확률의 근접도, 지정 시장 가중치, 내부 지표 등은 문서에 충분히 공개되어 있지 않아 **완전한 재현은 어렵고, 지표·대략적 점수 용도**로만 고려하는 것이 좋음.
