# Cursor 협업 규칙 (Claude용)

**당신(Claude)은** 코드 리뷰와 PR 검토·작성을 담당합니다. 코드 작성·구현·리팩터링은 Cursor가 합니다.

전체 워크플로우는 **`docs/COLLAB_WORKFLOW.md`**에 정의되어 있습니다. Git에 올라가는 단일 기준이므로, 워크플로우가 헷갈리면 그 파일을 먼저 읽으세요.

---

## 당신이 할 일

1. **입력:** 사용자가 "Cursor로 작업한 최신 브랜치에서 diff만 보고 ~ 해 줘"라고 하면, **해당 브랜치의 `git diff`**를 기준으로 작업합니다.
2. **리뷰/PR:** 그 diff만 보고 코드 리뷰 또는 PR 설명/검토를 수행합니다. 전체 코드베이스를 몰라도, diff만으로 판단 가능하도록 Cursor 쪽 규칙이 맞춰져 있습니다.
3. **브랜치명:** Cursor 작업 완료 요약에 브랜치명이 포함되므로, 사용자가 지정한(또는 최신) 작업 브랜치의 diff를 사용하면 됩니다.

---

## Cursor와 맞춰 둔 것

- Cursor는 구현 시 **주석**을 넣어 두어, diff만 봐도 맥락을 알 수 있게 합니다. 리뷰 시 그 주석을 활용하세요.
- **한 커밋 = 한 논리적 단위**가 되도록 Cursor가 정리해 둡니다. 리뷰/PR 시 그 단위가 지켜졌는지 확인하면 됩니다.
- **README 등 문서** 변경은 Cursor가 하고, 리뷰 시 논리적 단위에 포함되었는지 확인합니다.

---

## 규칙 동기화

- `.cursorrules`, `.cursor/rules/`에도 같은 협업 규칙이 있습니다.
- **단일 기준:** `docs/COLLAB_WORKFLOW.md`. 내용이 어긋나면 그 문서를 기준으로 맞추세요.
